# Data Contract: questions_by_date.csv

## Назначение
Датасет содержит агрегированные по дням данные о количестве новых вопросов
на Stack Overflow для выбранных технологических тегов.  
Является основным источником данных для анализа трендов и популярности.

---

## Временная гранулярность
Календарный день.

---

## Таймзона
Все даты приведены к **UTC**.

Поле `date` соответствует дате создания вопроса (`creation_date`),
преобразованной в UTC и агрегированной до календарного дня.

---

## Схема данных

| Поле | Тип | Формат | Описание |
|------|-----|--------|----------|
| `date` | date | YYYY-MM-DD | Дата создания вопросов (UTC) |
| `tag` | string | enum | Технологический тег |
| `questions_count` | integer | ≥ 0 | Количество новых вопросов за день |

---

## Допустимые значения

### `tag`
Допустимые значения (фиксированный регистр):
- `Python`
- `Java`
- `Javascript`

Использование других значений не допускается.

---

## Правила агрегации
- Каждый вопрос учитывается один раз в рамках соответствующего тега.
- Вопросы с несколькими тегами учитываются для каждого соответствующего тега.
- Агрегация выполняется по дате создания вопроса.
- Пропущенные даты заполняются нулевыми значениями.
- Дубликаты строк с одинаковыми (`date`, `tag`) не допускаются.

---

## Гарантии качества данных
- Отсутствуют пропущенные значения.
- Поле `questions_count` является неотрицательным целым числом.
- Для каждой даты присутствует ровно одна запись для каждого тега.
- Кодировка CSV: UTF-8.
- Разделитель столбцов: запятая (`,`).

---

## Пример данных

```csv
date,tag,questions_count
2025-07-01,Python,134
2025-07-01,Java,98
2025-07-01,Javascript,120
```
